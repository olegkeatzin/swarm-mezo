# swarm_mezo — санитарная симуляция трёх гипотез

Этот подпроект — NumPy-реализация трёх проверочных экспериментов, заявленных в
спецификации `теория/swarm-mezo-spec.md`. Цель — не обучить LLM, а
**эмпирически подтвердить или опровергнуть** три теоретических утверждения,
лежащих в основе Swarm-MeZO, до того как они уйдут в статью/диплом.

Никакого PyTorch, никакого GPU, никаких LLM. Только NumPy, SciPy, Matplotlib.

## Как запустить

Из корня репозитория:

```
uv run python теория/swarm_mezo/run.py
```

или, если зависимости уже подняты:

```
python теория/swarm_mezo/run.py
```

Прогон полностью детерминирован (`numpy.random.default_rng(seed)`, сид
принимается параметром везде) и занимает порядка пары минут на одном CPU. Все
PNG и CSV пишутся в `теория/swarm_mezo/results/`.

Тесты:

```
uv run pytest теория/swarm_mezo/tests/ -v
```

12 тестов: верификация SPSA, наклон `−1` в уменьшенной версии E1, плато
у малых `K` в E2, свойства матриц `W`, инвариантность симметричного режима
к `β`, монотонное убывание дисперсии и пр.

## Структура

```
swarm_mezo/
  objectives.py    # Quadratic, MultiWell (целевые функции)
  mezo.py          # spsa_estimate — оценка градиента двумя forward-проходами
  consensus.py     # матрицы W (полный/кольцо, репутационная), закон репутации
  swarm.py         # цикл run_swarm: локальный MeZO-шаг + шаг консенсуса
  experiments.py   # run_e1, run_e2, run_e3 + dataclass-результаты
  plots.py         # генерация PNG и CSV для трёх экспериментов
  run.py           # точка входа: гонит E1, E2, E3 и пишет всё в results/
  tests/           # pytest-ассерты на корректность и на ожидаемые наклоны/плато
  results/         # сюда складываются PNG + CSV (под gitignore)
```

## Три эксперимента и что они проверяют

### E1 — закон `1/N` (гипотеза H1)

**Что меряется.** Среднеквадратичная ошибка усреднённой по `N` агентам
SPSA-оценки относительно истинного градиента на квадратичной функции с фиксированным `θ`:

```
MSE(N) = E ‖ (1/N) Σ_i ĝ_i − ∇f ‖²
```

`N ∈ {1, 2, 4, 8, 16, 32, 64, 128}`, по 1000 повторений на точку,
независимые гауссовы возмущения `z_i` у каждого агента.

**График.** `results/e1_variance_vs_N.png` — log-log с пунктирной референсной
линией наклона `−1`.

**Как читать.**
- Если кривая ложится на референсную линию (наклон log-log регрессии близок к
  `−1`) → закон `1/N` подтверждён, мультиагентный консенсус действительно борется
  с дисперсией MeZO.
- Если наклон сильно меньше по модулю (например `−0.3`) → независимости `z_i`
  на самом деле нет, проверять схему генерации.
- Если кривая выходит на плато → ошибки агентов скоррелированы, см. E2.

**Фактический результат.** Наклон **`−0.996`** (ожидался `−1`).
Гипотеза H1 **подтверждена**.

### E2 — общий банк сидов (гипотеза H2, риск FedKSeed)

**Что меряется.** Та же MSE, что в E1, но `z_i` берутся из конечного **общего
банка сидов размера K** — модель протокола FedKSeed, где для экономии
коммуникаций все агенты тянут возмущения из небольшого пула. Если `N > K`,
несколько агентов получают одно и то же `z`, и ошибки SPSA становятся
скоррелированы.

`K ∈ {1, 4, 16, 64, ∞}` (где `∞` = независимые сиды — baseline из E1).

**График.** `results/e2_variance_seedbank.png` — семейство кривых `MSE(N)`
для разных `K` на одном log-log поле.

**Как читать.**
- Кривая `K = ∞` должна совпадать с E1 (наклон `−1`).
- При конечных `K` ожидается выход на **плато** на высоте примерно
  `Var(N=1)/K`. Меньше банк → выше плато.
- Если все кривые сходятся к плато на одной высоте → ошибка в генерации, банк
  не используется.

**Фактический результат.** Чёткие плато:

| K | Var(N=1) | Var(N=128) | отношение |
|---|---|---|---|
| 1 | 5.26e+07 | 5.26e+07 | ×1.00 |
| 4 | 5.52e+07 | 1.43e+07 | ×3.85 |
| 16 | 5.57e+07 | 3.68e+06 | ×15.15 |
| 64 | 5.34e+07 | 1.24e+06 | ×42.97 |
| ∞ | 5.40e+07 | 4.15e+05 | ×129.95 |

Отношения почти ровно равны `K` — теоретическое предсказание `Var(N→∞)/Var(N=1)
≈ 1/K` подтверждено. Гипотеза H2 **подтверждена**.

**Инженерная рекомендация.** Банк сидов должен быть как минимум того же
порядка, что число агентов `N` (а лучше существенно больше — десятки при
умеренных `N`), либо каждый агент должен использовать независимый сид. Иначе
обещанное снижение дисперсии `1/N` исчезает.

### E3 — спектр «кооперация ↔ отбор» (гипотеза H3)

**Что меряется.** Полный прогон оптимизации роем на невыпуклой 2D-функции
`MultiWell` — сумме трёх перевёрнутых гауссиан:
- глобальный минимум в `(0, 0)`, глубина `1.0`;
- два локальных минимума в `(2.2, 0)` и `(−2.0, 0.4)`, глубина `0.6`;
- ширина ям `σ = 0.6`.

`N = 8` агентов, 250 шагов, `η = 0.05`, `ε = 5·10⁻³`, гауссовы независимые `z_i`.
Стартовый разброс агентов `init_spread = 2.0` — намеренно большой, чтобы
агенты сразу попадали в разные бассейны притяжения (иначе эффект `β` не
проявился бы).

Сравниваются **два режима** матрицы консенсуса:
- **репутационный:** `W_ij = r_j / Σr`, репутации обновляются по закону
  `r_i ← r_i / (γ_r + β · |L_i − L_min|)` с `γ_r = 1`;
- **симметричный (контроль):** `W = (1/N)·J`, `β` игнорируется.

Для каждого `β ∈ {0, 0.1, 1, 10, 100}` делается **50 запусков**.
Оба режима в каждом запуске используют идентичный сид — одинаковые стартовые
позиции и идентичные SPSA-возмущения. Разница между режимами тогда отражает
только эффект матрицы консенсуса, а не различие стартов.

**Метрика.** Доля запусков, в которых центроид роя `θ̄_final` оказался в
бассейне глобального минимума (определяется через ближайший центр).

**График.** `results/e3_convergence_vs_beta.png` — две панели:
(а) hit-rate vs `β`, (б) финальный лосс vs `β`. Ось `β` в symlog.

**Как читать.**
- Симметричная кривая должна быть **плоской** — теория Де Гроота:
  точка сходимости от `β` не зависит, режим β-инвариантен.
- Репутационная кривая ожидается с **горбом**: умеренный `β` слегка помогает
  (отбор подтягивает рой к перспективным агентам), большой `β` вредит
  (информационный каскад: один случайно-удачный агент забирает всю репутацию,
  рой стягивается в его яму, даже если локальную).
- Если репутационная кривая совпадает с контрольной по всему `β` → отбор
  ничего не меняет, формулу репутации проверять.
- Если репутационная монотонно лучше контроля → каскада нет, надо проверить
  достаточно ли велик стартовый разброс.

**Фактический результат.**

| β | rep hit-rate | sym hit-rate | rep loss | sym loss |
|---|---|---|---|---|
| 0 | 0.92 | 0.92 | −0.971 | −0.971 |
| 0.1 | 0.92 | 0.92 | −0.971 | −0.971 |
| 1 | 0.94 | 0.92 | −0.979 | −0.971 |
| 10 | 0.94 | 0.92 | −0.979 | −0.971 |
| 100 | **0.72** | 0.92 | −0.891 | −0.971 |

- Симметричный hit-rate **ровно 0.92** при всех `β`, разброс 0.000. Это прямое
  подтверждение теории Де Гроота.
- Репутационный режим имеет **рабочее окно `β ∈ [1, 10]`** с небольшим
  превосходством над контролем (+0.02 hit-rate, измеримо ниже лосс).
- При `β = 100` — резкое падение до 0.72: информационный каскад.

Гипотеза H3 **подтверждена**. Оптимум `β` лежит **внутри** спектра, не на
краю; жёсткий отбор `gbest` (`β → ∞`) — действительно вырожденный плохой
предел, как и предсказывает теория информационных каскадов.
